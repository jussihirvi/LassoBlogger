Welcome to LassoBlogger 0.9.2.

Where to find it?

Since 0.9.2, LassoBlogger can be downloaded at Github. If you want to see older versions, see LassoForge.

Installation (upgraders, please scroll down)

  1. Unpack the zip file in your web directory
  2. Open up config-sample.lasso with a text editor (on Windows, use WordPad or similar) and fill in your database details (db name, username and password). Check the rest of the file, too (you can set your language preference etc.)
  3. Save the file as config.lasso
  4. Launch /admin/install.lasso in your browser and follow it's directions. This should help you setup the database and tables needed for your blog. If there is an error, double-check your config.lasso file, and try again. If it fails again, please read the dev blog or contact the developer - don't just give up!
  5. Note the password given to you by the install script.
  6. When finished, the install script should show you a link to the admin pages for login. Sign in with the username admin and the password generated during the installation. You can then click on 'Users', etc., to change the password.
  7. After installation, it's a good idea to hide /admin/install.lasso from malicious users or remove it completely - though there is some built-in protection in the script against repeated use.

Upgrading from older versions

If you have an older version of LassoBlogger, first

  • it's a good idea to back up your blog database first (for exmaple you can duplicate in in the DatabaseBrowser.lassoapp), so you will be able to go back to a previous version in case something goes wrong with the upgrade
  • remember to back up your old blog files, too - below is one way to do it:
    • move the zip file next to your active blog directory and unpack it
    • copy your old config.lasso from your active blog directory to the new version
    • do the same to any other files that you may have customized (customization of the style definitions in gen/frame_default.inc is recommended, customization of themes/default/style.css is *not* recommended, as it may change from version to version)
    • rename your old blog directory, for example from blog to blog_OLD (you can delete it later, when you are sure you have resqued from it everything you need)
    • rename the new blog directory (the one you just unzipped) with the original name, for example blog (you should use the same name as before, otherwise your blog's www address will change)

From v0.7.3 up, you can check your LassoBlogger version from the Admin Dashboard page. The version number is stored in the file inc/library.inc, right at the top.

If you upgrade from 0.5, run admin/upgrade_05_06.lasso to upgrade to 0.6. Then follow directions below.

If you upgrade from 0.6, open the new config_sample.lasso, scroll down, copy the last line (Variable:'start'...) and place it at the bottom of your active copy of config.lasso. Then follow directions below.

If you upgrade from a version between 0.6 and 0.7.2, run admin/upgrade_06_073.lasso to upgrade to 0.7.3, then follow directions below.

If you upgrade from 0.7.3, run admin/upgrade_073_08.lasso to upgrade to 0.8.0, then follow directions below.

If you upgrade from 0.8.0b1 (or any older version), you have to copy the variable dbConfig from config_sample.lasso to your active config.lasso. Then follow directions below.

If you upgrade from 0.9.0, please run admin/upgrade_090_091.lasso.

If you upgrade from 0.9.1, please run admin/upgrade_091_092.lasso.

After running any upgrader scripts, it is a good idea to hide them from malicious users.

System Recommendations

  • Lasso Professional 8 or newer (may work on older version, but is not tested; I have developed this on L6.0.6 until v0.8.0b1; then on L8.5, and now on L9.1.4).
  • MySQL; may work on other SQL db's, but is not tested

New in 0.9.2 - Lasso 9 compatibility, scale & rotate images

Now compatible with Lasso 9. Phew, that was a lot of work.

Redesigned admin area with new, faster navigation.

Scale and/or rotate images when uploading.

Better code. Practically all code has been modernized (colons -> parens, to start with), and a lot of refactoring has been done. The footprint is now much smaller.

CKeditor instead of FCKeditor on admin pages.

New in 0.9.1 - FCKeditor, and better spam control

FCKeditor for admin pages. For some instructions, see the Version History.

New spam-filtering feature based on an encrypted timestamp on the commenting form. This feature is built-in and cannot be adjusted by the means of Options.

Image links: possibility to make an uploaded image to work as a link. This can be done on the admin page Uploads > Manage Files. For some instructions, see the Version History.

Uploaded images can now be tied to a post manually, after upload. This can be done on the admin page Uploads > Manage Files.

For bug fixes: See the version history.

New in 0.9.0 - table prefixes, blog by email

Table prefixes - many apps per single database: a new variable in config.lasso facilitates sharing one single database between several LB blogs or other applications. If for example the table prefix is set to ”bl1_”, the posts table will be stored as bl1_posts, and all other blog tables likewise.

Send articles to your blog via email: if you set up a dedicated email address to your blog, and configure the relevant options, and set up a Lasso event to fetch email from your email server, you can start blogging by sending emails to your blog. Handy when you only have a slow mobile internet connection. For a basic documentation see this short article. For a general tutorial see this article from the WordPress codex.

Graphical smilies: a small but nice feature. Switchable through Writing Options.

Login/Logout links in the public view: always visible, when the user is not already logged in.

Other features & fixes: See the version history.

New in 0.8 - file uploads and other features

0.8.0: : A new Quick Delete button on admin page Manage > List Comments. Saves time, in case your blog is ever flooded with spam.

0.8.0: : On admin pages, a warning text will be displayed, if the user tries to delete a non-empty post-category or file-category record.

0.8.0b1: Options have been regrouped, so now the Admin > Option pages should be a little easier to understand.

0.8.0b1: A new option multiuser_blog should provide an easy way to choose between one-author and multi-author blogs - this option is however not activated as of this writing.

New in 0.7.3 - comment control

Options related to commenting have now been activated. You will find them on the admin page Options > Discussion, except for comment_registration, which for historical reasons is on the page Options > General.

Now LassoBlogger should have at least elemental protection against spam comments. For an overview of how to control commenting, see the file lb_commenting_options.txt in the docs.

New in 0.7.2

Categories can now be sorted (newest-to-oldest, or vice versa). A new field cat_sortorder is created for this purpose by the upgrader script.

New in 0.7 - permalink and languages

The two big new features are permalinks and language support.

When using permalinks, links to your post may look like www.MyDomain.org/MyBlog/2005/12/My_Post_Title.

Permalink structure can be modified by adjusting the related option (new in this version) on the admin page Options > Permalinks. Permalinks use the "error.lasso method". This required that you put a modified error.lasso file to your domain root, where Lasso will use it in case of any errors.

Check the enclosed error.lasso. you can use it as such, or modify (for example, you could add link to webmaster, or company logo). Then put it to the domain root.

Of course, you can perfectly well use LassoBlogger without using permalinks. In that case links to your posts look like this: www.MyDomain.org/MyBlog/?p=15.

The language support works this way:

  • first try to find out if a language file for your language already exists; take a look to the languages folder; read the official dev blog; ask the developer
  • if not, it's easy to do - about 70-80 phrases to translate from English, and they all are in one file; it took me about 30 minutes to do the Finnish translation (it's all in languages/fi.inc)
  • decide upon a language code (for example de, fr, or en_UK)
  • open languages/language_template.inc and follow directions in it
  • finally, activate your translations by changing the value of variable lang from "en" to your chosen code (in config.lasso)

New in 0.7.0 is also the possibility to create static pages.

Fighting spam comments

From version 0.9.1, the main line of defence is based on an encrypted timestamp on commenting form. If the timestamp is missing, or if it shows that too much time has elapsed since the form was loaded, the comment is discarded.

From version 0.7.3 there are a number of options available to regulate commenting.

There are so many options that it's easy to get things mixed up. I recommend the following:

  • start with all comments allowed (check your discussion options)
  • read docs/lassoblogger_info/commenting_options.txt to get an overview of available options, so you will be ready to act
  • if and when you get first spam comments, start tightening the rules; start from the least powerful options. Go as far as needed, until the spam stops but the good comments keep coming

Customising the layout

The basic stylesheet is themes/default/style.css. For v0.7, I moved style definitions that affect the page header (the top of page, where you see blog name and description) from the stylesheet to the top of the html template, gen/frame_default.inc. You can adjust them to your liking, and on later upgrades be sure to backup and reuse your modified frame_default.inc. I may modify the main stylesheet from version to version, though I try to be conservative about it.

Of course you can also revamp the stylesheet - that equals creating a new theme, and I'm interested to distribute LB themes. Besides a stylesheet, a theme can also include images.

User Levels

The admin page "Edit users" helps you grade the levels. The theoretical levels range from 0 to 10, but essentially the relevant levels are:

  • 0 - Can edit his/her own user info
  • 1 - Can post, edit and delete drafts (posts that need administrator approval to get published)
  • 2 - Can post, edit, and delete his/her own posts
  • 5 - May edit links and categories; sees list of users
  • 8 - Admin - may edit options, and edit posts, comments and user info of all users
  • 10 - Max value

In the usual case you may want a team of level 2 users (writers) to work with you (the administrator). You may also want one or two assistant admins (level 8 would be good for that).

Past and future

Since May 2009, LassoBlogger code was maintained at Google Code. In April 2012, it was moved to Github.

I hope the Github repository would encourage other developers to participate. If you cannot find your favorite feature in LassoBlogger, please code it yourself and submit the results to Github repository – or make a feature/fix request to me.
